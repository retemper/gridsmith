# Accessibility

Gridsmith implements the WAI-ARIA 1.2 grid pattern with screen-reader announcements.

## Grid pattern

| Element                | Role / attribute                                    |
| ---------------------- | --------------------------------------------------- |
| Grid root              | `role="grid"` + `aria-rowcount` + `aria-colcount`   |
| Header container       | `role="rowgroup"`                                   |
| Header row             | `role="row"` + `aria-rowindex` (1-based)            |
| Header cell            | `role="columnheader"` + `aria-colindex`             |
| Group header cell      | `aria-colspan` / `aria-rowspan`                     |
| Body row               | `role="row"` + offset `aria-rowindex`               |
| Cell                   | `role="gridcell"` + stable `id` + `aria-colindex`   |
| Read-only column cells | `aria-readonly="true"`                              |
| Cell with error        | `aria-invalid="true"`                               |
| Sorted header          | `aria-sort="ascending"` or `aria-sort="descending"` |

`aria-rowcount` includes header rows + pinned top + data rows + pinned bottom.

## Container-focus model

- The **grid root** holds keyboard focus.
- `aria-activedescendant` points at the focused cell's `id`.
- Cell `id`s are stable — generated by `buildCellId(gridId, rowIndex, columnId)`.

The format produced by `buildCellId` and `buildPinnedCellId` is part of the public contract — future format changes will ship as a breaking change.

This model is faster (no per-cell focus events) and screen readers handle it well.

## Live regions

Each grid creates two live regions next to the root:

- A polite region (`role="status"`) — sort / filter summaries, paste dimensions.
- An assertive region (`role="alert"`) — validation errors.

A 150 ms debounce prevents announcement storms during fast typing or rapid sorts.

`<Grid />` mounts the live regions internally and wires grid events to them — no extra hook required. For non-React adapters, the lower-level `createAnnouncer()` is exported from `@gridsmith/core`.

## SSR

The React adapter derives the grid `id` from `useId()`, so server-rendered cell IDs match on hydration.

## Color and contrast

The default theme passes WCAG AA. If you ship a custom theme, verify:

- Active cell border vs. cell background.
- Selected cell background vs. unselected.
- Validation error indicator (`gs-cell--invalid`) vs. background.

## Helpers (core)

For non-React adapters or custom renderers:

```ts
import {
  buildCellId,
  buildPinnedCellId,
  computeRowCount,
  createAnnouncer,
  dataRowAriaIndex,
  headerRowAriaIndex,
  nextGridInstanceId,
  pinnedBottomAriaIndex,
  pinnedTopAriaIndex,
} from '@gridsmith/core';
```

These produce identical IDs and indices to what the built-in renderer emits.
